  <behavior>
The Bradley-Terry model estimates relative strengths from pairwise comparisons:
  P(i beats j) = pi_i / (pi_i + pi_j)

**Test Cases:**
1. Simple 2-athlete case: A beats B 3 times, B beats A 1 time
   - A's strength should be higher than B's
   - Probability P(A beats B) should be ~0.75

2. 3-athlete case: A > B > C (transitive)
   - Rankings should be A, B, C in order
   - P(A beats C) > P(A beats B)

3. Sparse data: A vs B only, B vs C only
   - Should still produce rankings via transitivity
   - A vs C should have lower confidence

4. Edge case: All ties (everyone beats everyone equally)
   - All strengths should be approximately equal
   - All probabilities ~0.5

5. Standard errors increase with fewer comparisons
  </behavior>
  <implementation>
Use fmin library for Nelder-Mead optimization of negative log-likelihood.

Key functions:
- fitBradleyTerryModel(comparisons) → { athletes: [{athleteId, strength, stdError, ci}], convergence }
- computeProbabilityMatrix(strengths) → P[i][j] matrix
- computeStandardErrors(hessian, solution) → standard errors via Fisher information

Identifiability constraint: Sum of log-strengths = 0 (normalization)

Handle edge cases:
- Empty comparisons → error
- Single athlete → return default
- Disconnected graph → warning + fit connected components separately
  </implementation>
</feature>
