Create the three-level design token system in `src/v2/styles/tokens.css`:

**Level 1 - Base Palette:** Raw color values matching existing Tailwind config colors but as CSS variables:
- `--palette-void-deep: #08080A`
- `--palette-void-surface: #0c0c0e`
- `--palette-void-elevated: #121214`
- `--palette-text-primary: #F4F4F5`
- `--palette-text-secondary: #A1A1AA`
- `--palette-text-muted: #71717A`
- `--palette-blue: #0070F3`
- `--palette-violet: #7C3AED`
- (Include full spectrum from existing tailwind.config.js)

**Level 2 - Semantic Tokens:** Context-aware mappings:
- `--color-bg-surface: var(--palette-void-surface)`
- `--color-bg-elevated: var(--palette-void-elevated)`
- `--color-text-primary: var(--palette-text-primary)`
- `--color-text-secondary: var(--palette-text-secondary)`
- `--color-action-primary: var(--palette-blue)`
- `--color-border-default: rgba(255, 255, 255, 0.08)`
- etc.

**Theme Overrides:** Add `[data-theme="light"]` and `[data-theme="field"]` selectors that remap semantic tokens:

```css
.v2[data-theme="light"] {
  --color-bg-surface: #FFFFFF;
  --color-bg-elevated: #F9FAFB;
  --color-text-primary: #18181B;
  --color-text-secondary: #52525B;
  --color-action-primary: #0070F3;
}

.v2[data-theme="field"] {
  --color-bg-surface: #000000;
  --color-bg-elevated: #0a0a0a;
  --color-text-primary: #FFFFFF;
  --color-text-secondary: #E4E4E7;
  --color-action-primary: #3B82F6;
}
```

Create `src/v2/styles/v2.css` that imports tokens and Tailwind:
```css
@import './tokens.css';

/* V2-scoped Tailwind - loaded by V2Layout */
@tailwind base;
@tailwind components;
@tailwind utilities;
```

**Important:** Dark mode is the default (no data-theme attribute needed). Light and field modes require explicit `data-theme` attribute.

  <name>Task 3: Create V2 Tailwind Config</name>

  <action>
Create `tailwind.v2.config.js` as a separate config for V2:

```javascript
/** @type {import('tailwindcss').Config} */
export default {
  important: '.v2',  // Scope ALL utilities under .v2 selector
  content: [
    "./src/v2/**/*.{js,ts,jsx,tsx}",  // Only V2 files
  ],
  darkMode: 'class',
  theme: {
    extend: {
      colors: {
        // Map Tailwind classes to CSS custom properties
        'bg-surface': 'var(--color-bg-surface)',
        'bg-elevated': 'var(--color-bg-elevated)',
        'text-primary': 'var(--color-text-primary)',
        'text-secondary': 'var(--color-text-secondary)',
        'text-muted': 'var(--color-text-muted)',
        'action-primary': 'var(--color-action-primary)',
        'border-default': 'var(--color-border-default)',
      },
      // Inherit typography, spacing, shadows from main config
      fontFamily: {
        'display': ['"Space Grotesk"', 'system-ui', 'sans-serif'],
        'sans': ['"DM Sans"', 'system-ui', 'sans-serif'],
        'mono': ['"JetBrains Mono"', 'ui-monospace', 'monospace'],
      },
    },
  },
  plugins: [],
}
```

Update `vite.config.ts` to add path alias for V2:
- Add `'@v2': path.resolve(__dirname, './src/v2')` to resolve.alias

Update `postcss.config.js` if needed to process V2 styles (typically no changes needed as Vite handles this automatically).

**Critical:** Do NOT modify the existing `tailwind.config.js` - V1 must remain unchanged. The V2 config is loaded via V2Layout's CSS import.
