**Prerequisites:**
- Dev server running (npm run dev on port 3001)
- Logged in as a coach or owner role user
- Navigate to http://100.86.4.57:3001/beta

**1. Coach Context Navigation**
- Click "Coach" in the left rail
- Verify sidebar shows: Whiteboard, Fleet, Availability items
- Click each item to navigate

**2. Whiteboard Feature (COACH-01, COACH-02)**
- Navigate to /beta/coach/whiteboard
- If no whiteboard exists: Verify empty state with "Create" button
- Click Create/Edit button
- Verify markdown editor appears with live preview
- Enter markdown content (try headers, lists, bold)
- Click "Post Whiteboard"
- Verify content displays with rendered markdown
- Verify date and author shown

**3. Fleet Management (COACH-03, COACH-04)**
- Navigate to /beta/coach/fleet
- Verify tabs for Shells and Oars
- Click "Add Shell" button
- Verify form with fields: name, boat class, type, weight, rigging, status, notes
- Try submitting empty form - should show validation errors
- Fill form and submit
- Verify shell appears in table with status badge
- Click edit icon - verify form pre-fills
- Click delete icon - verify confirmation and removal

- Switch to Oars tab
- Click "Add Oar Set"
- Verify form with: name, type (Sweep/Scull), count, status, notes
- Test CRUD operations same as shells

**4. Athlete Biometrics Display (COACH-05)**
- Navigate to /beta/coach/availability
- Look at the athlete name column (leftmost)
- Verify each athlete shows biometrics badges next to their name:
  - Side preference badge: P (Port), S (Starboard), B (Both), C (Cox)
  - "Sc" badge if athlete can scull
  - "Cx" badge if athlete can cox
- If no athletes have biometrics set, update an athlete's profile to add:
  - Set side preference
  - Set canScull = true
  - Set canCox = true
- Refresh and verify badges appear

**5. Availability Feature (COACH-06, COACH-07)**
- In /beta/coach/availability
- Verify grid shows athletes (rows) x dates (columns)
- Verify week navigation arrows and "This Week" button
- Navigate to different weeks
- Verify color coding: green=available, red=unavailable, yellow=maybe, gray=not set
- Click on an athlete name
- Verify editor modal opens with date selectors
- Change some slots and save
- Verify grid updates

**6. Role-Based Access**
- If testing as athlete (not coach):
  - Whiteboard: should see view only, no edit button
  - Fleet: should see tables but no add/edit/delete buttons
  - Availability: should only be able to edit own availability

**Success Criteria:**
- All three coach pages load without errors
- Whiteboard CRUD works with markdown
- Fleet CRUD works with validation
- Availability grid displays biometrics badges correctly
- Availability editing works
- No console errors
