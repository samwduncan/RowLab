Create a generic virtualized table component with these features:

Props interface:
```typescript
interface Column<T> {
  key: keyof T | string;
  header: string;
  width?: number | string;
  sortable?: boolean;
  render?: (item: T, index: number) => React.ReactNode;
}

interface VirtualTableProps<T> {
  data: T[];
  columns: Column<T>[];
  rowHeight?: number; // default 48
  maxHeight?: number | string; // default 600
  onRowClick?: (item: T) => void;
  sortColumn?: string;
  sortDirection?: 'asc' | 'desc';
  onSort?: (column: string) => void;
  emptyMessage?: string;
  getRowKey: (item: T) => string;
}
```

Implementation:
1. Use useVirtualizer from @tanstack/react-virtual
2. Virtualize rows but render all columns (column count is small)
3. Support sortable column headers with ChevronUp/ChevronDown icons from lucide-react
4. Apply V2 design tokens for styling (txt-primary, txt-secondary, bdr-primary, surface-hover)
5. Handle empty state with emptyMessage prop
6. Support hover and click states on rows
7. Use sticky table header

Key implementation details:
- parentRef for scroll container
- estimateSize returns rowHeight
- Overscan 5 rows for smooth scrolling
- Translate rows using transform for GPU acceleration
- Table header outside virtualized area (sticky)

Follow existing V2 component patterns (see ShellsTable.tsx for styling reference).
Use Tailwind classes matching V2 design system.
