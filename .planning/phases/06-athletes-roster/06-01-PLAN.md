  <action>
Add a new Attendance model after the Athlete model with these fields:
- id: String @id @default(uuid())
- teamId: String (foreign key to Team)
- athleteId: String (foreign key to Athlete)
- date: DateTime (the practice/event date)
- status: String (PRESENT, LATE, EXCUSED, UNEXCUSED)
- notes: String? (optional reason/comment)
- recordedById: String? (coach who recorded, foreign key to User)
- createdAt: DateTime @default(now())
- updatedAt: DateTime @updatedAt

Add relations:
- team: Team @relation(fields: [teamId], references: [id], onDelete: Cascade)
- athlete: Athlete @relation(fields: [athleteId], references: [id], onDelete: Cascade)
- recordedBy: User? @relation(fields: [recordedById], references: [id])

Add index: @@index([teamId, date])
Add unique constraint: @@unique([athleteId, date]) (one record per athlete per day)
Map to table: @@map("attendance")

Also update Athlete model to include: attendance Attendance[] relation.
Also update User model to include: attendanceRecords Attendance[] relation.

Run `npx prisma db push` to apply changes (no migration needed for development).

<task type="auto">
  <name>Task 2: Create attendance service layer</name>
  <files>server/services/attendanceService.js</files>
  <action>
Create new service file with these functions:

1. recordAttendance(teamId, data): Record/update attendance for athletes
   - data: { date: string, records: [{ athleteId, status, notes? }] }
   - Uses upsert to handle updates (same athlete + date = update)
   - Returns { success: true, recorded: count }

2. getAthleteAttendance(teamId, athleteId, options): Get attendance history for one athlete
   - options: { startDate?, endDate?, limit? }
   - Returns array of attendance records with dates and statuses
   - Includes summary stats: { present, late, excused, unexcused, total }

3. getTeamAttendance(teamId, options): Get team-wide attendance
   - options: { date?, startDate?, endDate? }
   - If date provided: returns all athletes with their status for that date
   - If range provided: returns summary by athlete (attendance rates)
   - Includes team-level stats

4. deleteAttendance(teamId, id): Delete a single attendance record
   - Verify record belongs to team before deleting

Use Prisma for all queries. Follow existing service patterns (see athleteService.js).
Import prisma from '../db/connection.js'.
Export all functions as named exports.
  </action>
  <verify>node -c server/services/attendanceService.js</verify>
  <done>Service file exists with all four functions, no syntax errors</done>

<task type="auto">
  <name>Task 3: Create attendance API routes</name>
  <files>server/routes/attendance.js, server/index.js</files>
  <action>
Create new route file with these endpoints:

POST /api/v1/attendance
- Body: { date: string, records: [{ athleteId, status, notes? }] }
- Validates date is ISO8601, status is valid enum, athleteId is UUID
- Calls attendanceService.recordAttendance
- Returns 201 on success

GET /api/v1/attendance
- Query: date (single day) OR startDate & endDate (range)
- Returns team attendance for date/range
- Calls attendanceService.getTeamAttendance

GET /api/v1/attendance/athlete/:athleteId
- Params: athleteId (UUID)
- Query: startDate?, endDate?, limit?
- Returns athlete's attendance history with stats
- Calls attendanceService.getAthleteAttendance

DELETE /api/v1/attendance/:id
- Params: id (UUID)
- Calls attendanceService.deleteAttendance
- Returns 204 on success

All routes use authenticateToken and teamIsolation middleware.
Use express-validator for input validation.
Follow error handling pattern from athletes.js.

Mount in server/index.js:
- Import: import attendanceRoutes from './routes/attendance.js'
- Mount: app.use('/api/v1/attendance', apiLimiter, attendanceRoutes)
  </action>
