The issue: Light and field themes defined with `.v2[data-theme="..."]` selector may not be applying due to CSS specificity or missing data-theme attribute.

Diagnosis steps:
1. Check if V2Layout applies data-theme attribute to the .v2 element
2. Verify the CSS selector specificity is correct
3. Check if tokens.css is imported in the correct order

Fix V2Layout.tsx:
Ensure the layout root element has both the .v2 class AND the data-theme attribute:

```tsx
import { useTheme } from '../hooks/useTheme';

export function V2Layout({ children }: { children: React.ReactNode }) {
  const { theme } = useTheme();

  return (
    <div className="v2" data-theme={theme}>
      {/* rest of layout */}
    </div>
  );
}
```

If ShellLayout is the actual wrapper being used, apply the fix there instead.

Fix tokens.css:
Ensure CSS variable overrides have sufficient specificity:

```css
/* Increase specificity for theme overrides */
.v2[data-theme="light"],
[data-theme="light"] .v2 {
  /* Light theme tokens... */
}

.v2[data-theme="field"],
[data-theme="field"] .v2 {
  /* Field theme tokens... */
}
```

Also add body-level theme support as fallback:
```css
body[data-theme="light"] {
  /* Light theme tokens */
}
```
